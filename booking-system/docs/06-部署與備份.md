# 06｜本機啟動、正式部署、備份與還原

## 一、本機開發啟動

需求：Node.js 22+、PostgreSQL 16+（本機或 Docker）。

```bash
cd booking-system
npm install

# 準備資料庫（例：docker run -d -p 5432:5432 -e POSTGRES_PASSWORD=dev postgres:16-alpine）
cp .env.example .env        # 填入本機 DATABASE_URL 與隨機密鑰
npx prisma migrate deploy   # 建立資料表
npm run db:seed             # 醫師/門診/班表/測試帳號（admin、counter1）
npm run dev                 # http://localhost:3000
```

- 前台：`http://localhost:3000`；後台：`http://localhost:3000/admin`
- 開發模式下 OTP 簡訊印在終端機（`SMS_PROVIDER=console`），驗證碼欄會自動帶入。
- 測試：`npm test`（需 `lihsin_booking_test` 資料庫，見 `tests/global-setup.ts`）。

## 二、正式部署（Docker Compose，診所自有主機或 VPS）

建議規格：1 vCPU / 2GB RAM 即足；Ubuntu 22.04+；網域 `booking.lhpedclinic.com.tw` 指向主機。

```bash
# 1. 取得程式碼（僅 booking-system 目錄即可獨立運作）
git clone <repo> && cd lihsin-clinic/booking-system

# 2. 設定環境變數
cp .env.example .env
#   - 產生密鑰：openssl rand -base64 48（三把各自不同）
#   - 設 POSTGRES_PASSWORD、APP_BASE_URL、LINE、簡訊帳號

# 3. 啟動（app + db + 每日備份 + 提醒排程）
docker compose up -d --build

# 4. 建立資料表與基礎資料
docker compose exec app npx prisma migrate deploy
docker compose exec app node node_modules/tsx/dist/cli.mjs prisma/seed.ts   # 測試環境
# 正式環境改用（不建立測試帳號）：
docker compose exec -e ADMIN_USERNAME=admin -e ADMIN_PASSWORD='安全密碼' \
  app node node_modules/tsx/dist/cli.mjs scripts/create-admin.ts

# 5. HTTPS 反向代理（擇一）。以 Caddy 為例（自動憑證）：
#   booking.lhpedclinic.com.tw { reverse_proxy 127.0.0.1:3000 }
```

首次登入後台後**必做**：
1. 修改管理員密碼（員工帳號頁）。
2. 啟用管理員兩步驟驗證（TOTP）。
3. 建立櫃檯人員帳號。
4. 依實際班表調整「排班管理」與「系統設定」（seed 預設兩位醫師全時段雙診）。

### 測試環境（正式上線前供診所人員試用）

另開一個目錄/主機用相同步驟部署，`.env` 使用**不同資料庫與密鑰**、`SMS_PROVIDER=console`、
LINE 用測試 channel 或留空。驗收表見 `docs/01`。正式與測試環境嚴禁共用資料庫。

## 三、備份與還原

### 自動備份

`docker compose` 內建 `backup` 服務：每日 03:30 `pg_dump -Fc` 到 `./backups/`，保留 30 天。
建議再將 `backups/` 同步到異地（另一台 NAS／加密雲端），例如 crontab：

```bash
30 4 * * * rsync -a /path/booking-system/backups/ user@nas:/backup/lihsin-booking/
```

### 手動備份

```bash
docker compose exec db pg_dump -U lihsin -Fc lihsin_booking > backups/manual_$(date +%Y%m%d%H%M).dump
```

### 還原

```bash
docker compose stop app reminder
docker compose exec -T db pg_restore -U lihsin -d lihsin_booking --clean --if-exists < backups/lihsin_booking_YYYYMMDD.dump
docker compose start app reminder
```

### ⚠️ 金鑰備份

`PII_ENCRYPTION_KEY`／`PII_HASH_KEY` 遺失＝證件號永久無法解密/比對。
請將 `.env` 離線影本（紙本或加密隨身碟）由院長保管；備份檔與金鑰**分開存放**。

## 四、升級

```bash
git pull
docker compose build app reminder
docker compose exec app npx prisma migrate deploy   # 先跑 migration
docker compose up -d
```

## 五、個資保存與封存

- 預約與病人資料屬醫療掛號紀錄，預設保存；保存年限依診所個資政策（`privacy.retention_years` 設定，預設 3 年）。
- 到期封存做法：以備份 dump 冷存後，由管理員於資料庫執行匿名化腳本（將姓名/電話/證件密文以雜湊替代）；
  執行前務必先備份。此作業屬年度維護，建議由維護廠商或工程師執行。
