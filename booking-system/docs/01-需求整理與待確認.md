# 01｜需求整理與待診所確認事項

> 立欣診所自有線上預約系統。本文件整理需求書全文的落地方式，以及**仍需院長／診所確認的設定**。
> 系統原則：所有程式碼、資料庫、第三方帳號（LINE、簡訊、主機）由立欣診所自行持有；不依賴 BookNow 或任何預約 SaaS。

## 一、範圍總覽

| 模組 | 內容 | 狀態 |
|---|---|---|
| 前台 | 七步驟預約流程、查詢/取消/改期、LINE 登入、手機 OTP、規則與個資告知 | ✅ 已實作 |
| 後台 | 今日預約總覽（雙醫師欄）、搜尋、代約/改期/報到/未到、排班與例外、病人與黑名單、員工帳號、系統設定、稽核紀錄 | ✅ 已實作 |

## 二、關鍵規則落地方式（摘要，詳見 04）

- **名額**：每醫師每 30 分鐘 1 名（可由後台加開）；雙診＝兩醫師各自 1 名，時段綁定醫師；「不限醫師」自動分配並平衡兩位醫師人數。
- **同日唯一**：同一病人（以證件號＋生日識別，非 LINE 帳號）同一日曆日僅 1 筆有效預約（待確認＋已確認＋已報到）；資料庫 partial unique index 強制。
- **7 天 3 筆**：任意連續 7 天內最多 3 筆有效預約，跨醫師、跨門診合併計算；上限與天數可於後台設定。
- **14 天滾動開放**：預設每日台灣時間 00:00 開放第 14 天；天數可設定。
- **未到黑名單**：未到累計超過 3 次（第 4 次）自動限制前台預約；門檻可設定；後台可解除／暫時解除／重設，需輸入原因並留稽核。
- **超賣防護**：預約建立/改期/取消皆為單一交易；時段列 `SELECT … FOR UPDATE` 序列化 + partial unique index 雙保險。

## 三、⚠️ 仍需診所（院長）確認的設定

上線前請逐項確認，均可於後台「系統設定」直接調整，程式碼不需改動：

1. **同日/當天可否預約**：目前預設「當天仍可預約尚未開始的時段」（截止前 30 分鐘）。是否改為只能預約明天以後？
2. **取消/改期截止時間**：預設看診前 2 小時。是否需要更長（例如前一天 21:00）？
3. **每 30 分鐘名額**：預設 1 名/醫師。是否有時段想預設開 2 名？
4. **7 天上限**：預設 3 筆／7 天。維持或調整？
5. **未到門檻**：預設累計 >3 次（第 4 次未到）自動限制。維持或調整？
6. **待確認 vs 直接確認**：目前一般門診預約成立即為「已確認」；「兒童發展篩檢」「減重特別門診」「過敏特別門診」預設需櫃檯審核（先為「待確認」）。是否正確？
7. **特別門診開放時段**：三種特別門診實際開放的星期／時段／醫師，請提供正式班表（seed 內為示意設定）。
8. **醫師唯讀帳號**：是否啟用（預設已支援，未建帳號）。
9. **簡訊供應商**：需選定台灣簡訊商（三竹/every8d/其他）並提供 API 帳號；目前為可替換模組＋開發用 console 供應商。
10. **LINE 官方帳號**：需以診所帳號建立 LINE Login channel 與 Messaging API channel（設定步驟見 `05-LINE串接設定.md`）。
11. **看診當日提醒**：預設開啟（後台可關）。發送時間預設當日 08:00、前一日提醒 19:00，是否調整？
12. **個資保存期限**：預設預約紀錄保存 3 年後可封存（僅提供機制與說明，實際年限請依診所個資政策決定）。
13. **前台文案**：所有可見文字（首頁提醒、規則頁、錯誤訊息）依內部合規制度，正式上線前需院長逐字核可。
14. **網域**：建議使用 `booking.lhpedclinic.com.tw` 子網域（部署說明以此為例）。

## 四、與現有官網的關係

- 本系統位於 repo `booking-system/`，為**獨立部署**的 Next.js + PostgreSQL 應用；官網維持 Netlify 靜態站不變。
- `_redirects` 第 5 節已加入 `/booking-system/*` 強制 404，避免 Netlify 把原始碼當靜態檔公開。
- 官網可於「線上預約」按鈕連到本系統網址；BookNow 停用時程由診所決定，兩者可並行過渡。
