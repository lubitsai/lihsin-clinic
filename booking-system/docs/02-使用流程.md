# 02｜前台與後台使用流程

## 一、民眾預約流程（前台）

```mermaid
flowchart TD
    A[預約首頁] --> B[步驟1：選擇門診類型]
    B --> C[步驟2：選擇醫師或不限醫師]
    C --> D[步驟3：選擇日期（未來14天）]
    D --> E[步驟4：選擇可預約時段]
    E --> F[步驟5：填寫病人資料]
    F --> G{手機驗證 / LINE 已登入}
    G -->|OTP 驗證碼| H[步驟6：確認預約內容]
    G -->|LINE 已綁定| H
    H --> I{後端規則檢查（交易內）}
    I -->|通過| J[步驟7：預約成功＋預約編號]
    I -->|同日已有預約| K[顯示同日限制訊息]
    I -->|7天已3筆| L[顯示公平性限制訊息]
    I -->|受限名單| M[顯示請致電診所訊息]
    I -->|名額已滿| N[回到步驟4重選時段]
    J --> O[LINE / 簡訊通知預約成立]
```

- 每一步顯示進度條，可返回上一步；送出使用一次性 token 防重複送出。
- 「不限醫師」由系統自動分配仍有名額的醫師，優先平衡各醫師人數。
- 所有規則檢查在**後端交易內**再次執行，前端檢查僅為體驗提示。

## 二、查詢／取消／改期

```mermaid
flowchart TD
    A[查詢我的預約] --> B{身分確認}
    B -->|LINE 登入（已綁定）| C[列出未來預約]
    B -->|證件號＋生日＋手機OTP| C
    C --> D{操作}
    D -->|取消| E[確認 → 交易內取消 → 通知]
    D -->|改期| F[選新時段 → 交易內原筆標記已改期＋建立新筆 → 通知]
    F --> G{重新檢查：名額/同日/7天/受限/醫師排班}
```

- 取消與改期受後台設定的截止時間限制（預設看診前 2 小時）。
- 改期為單一交易：新時段檢查失敗時原預約完全不動。

## 三、櫃檯日常流程（後台）

```mermaid
flowchart TD
    A[員工登入（管理員需 TOTP 2FA）] --> B[今日預約總覽]
    B --> C[報到 / 完成 / 未到 / 取消 / 改期 / 編輯]
    B --> D[搜尋病人或預約]
    D --> E[代病人建立預約（可覆寫限制，需理由）]
    C --> F{標記未到}
    F -->|累計超過門檻| G[自動加入預約限制名單]
    B --> H[列印當日表 / 匯出 CSV]
```

## 四、排班調整流程

```mermaid
flowchart TD
    A[固定週班表] --> B[產生每日時段]
    C[日期例外：休診/換班/加診/封鎖時段] --> B
    C --> D{修改的時段已有預約？}
    D -->|是| E[顯示受影響病人名單]
    E --> F[逐筆選擇：改期 / 換醫師 / 診所取消]
    F --> G[批次發送 LINE/簡訊通知]
    D -->|否| H[直接生效]
    G --> I[全部寫入稽核紀錄]
```

## 五、未到與黑名單

1. 櫃檯於總覽將預約標記「未到」→ 寫入 `no_show_records`。
2. 累計超過門檻（預設 3 次）→ 自動建立 `booking_restrictions`（AUTO_NO_SHOW）。
3. 受限病人前台預約時顯示中性訊息（不出現「黑名單」字樣）。
4. 管理員可解除／暫時解除（設定到期日）／重設次數，均需原因＋稽核紀錄。
5. 櫃檯仍可替受限病人代約，需輸入理由。

## 六、通知時機

| 事件 | LINE 推播 | 簡訊（無 LINE 時） |
|---|---|---|
| 預約成立 | ✅ | ✅ |
| 預約修改／改期 | ✅ | ✅ |
| 預約取消（病人／診所） | ✅ | ✅ |
| 前一日提醒（預設 19:00） | ✅ | ✅ |
| 當日提醒（預設 08:00，可關） | ✅ | 後台開關 |
| 臨時休診／調班 | ✅（批次） | ✅（批次） |

通知內容一律不含完整身分證字號與敏感醫療資訊。
