## 二、預約狀態機

```
PENDING(待確認) → CONFIRMED(已確認) → CHECKED_IN(已報到) → COMPLETED(已完成)
      │                │                    │
      ├→ CANCELLED_BY_PATIENT(病人取消)      │
      ├→ CANCELLED_BY_CLINIC(診所取消)       │
      ├→ RESCHEDULED(已改期→產生新預約)       │
      └→ NO_SHOW(未到)  ←──────────────────┘（過號未報到由櫃檯標記）
```

- **有效預約**（占名額、計入同日與 7 天限制）：`PENDING`、`CONFIRMED`、`CHECKED_IN`。
- **占用歷史名額**（不可再被預約走）：上列加 `COMPLETED`、`NO_SHOW`。
- 取消／已改期立即釋放名額。
- 所有狀態變更寫入 `appointment_status_history` 與 `audit_logs`。

## 三、預約規則（後端一律重新檢查）

| 規則 | 預設值 | 設定鍵（system_settings） |
|---|---|---|
| 滾動開放天數 | 14 天（每日 00:00 開放第 14 天） | `booking.open_days` |
| 當日截止 | 時段開始前 30 分鐘 | `booking.same_day_cutoff_minutes` |
| 同日上限 | 1 筆有效預約／日曆日 | `booking.daily_limit`（固定 1，保留鍵） |
| 7 天上限 | 任意連續 7 天 3 筆 | `booking.window_days` / `booking.window_max` |
| 未到限制門檻 | 累計 >3 次（第 4 次）自動限制 | `booking.no_show_threshold` |
| 取消/改期截止 | 看診前 120 分鐘 | `booking.cancel_cutoff_minutes` |
| 每 30 分鐘名額 | 1 名/醫師（時段可手動加開） | `booking.default_slot_capacity` |
| 當日提醒開關 | 開 | `notify.same_day_reminder` |
| 前日提醒時間 | 19:00 | `notify.day_before_time` |
| 當日提醒時間 | 08:00 | `notify.same_day_time` |
| 閒置登出 | 30 分鐘 | `security.staff_idle_minutes` |

### 名額與「不限醫師」分配

1. 時段一律綁定醫師；雙診＝同一時間兩位醫師各有自己的時段，各 1 名。
2. 不得因雙診將兩名病人放入同一醫師同一時段。
3. 「不限醫師」：列出該時間有剩餘名額的醫師 → 選**當日已預約數較少**者；平手時取顯示順序在前者。
4. 後台可封鎖單一時段（`is_blocked`）或手動加開名額（記錄操作者、時間、原因）。

### 病人識別

- 預約限制以「實際看診病人」為單位：`(證件類型, 證件號雜湊)` 唯一識別，生日交叉驗證。
- 同一 LINE 帳號可替多位家庭成員預約，各成員限制各自計算。
- 台灣身分證驗證檢查碼；居留證（新式統一證號）驗證檢查碼；護照僅格式檢查。

### 訊息文案（可見文字，上線前需院長核可）

- 同日重複：「您當天已有預約，無法重複預約其他門診或時段。如需更改，請先取消原預約或聯絡立欣診所。」
- 7 天超限：「為維護預約公平性，每位病人在 7 天內最多預約 3 個時段。如有特殊需求，請致電立欣診所。」
- 受限名單：「目前無法使用線上預約，請致電立欣診所，由櫃檯人員協助。」（前台不出現「黑名單」字樣）
- 身分探測防護:查詢失敗一律回覆「查無符合的預約資料，請確認輸入內容」，不透露特定證件號是否存在。
