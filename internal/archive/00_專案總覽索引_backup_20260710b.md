## 四、合規規則（每次必守）

1. 可見區禁自稱超級詞（推薦/最佳/第一/首選/權威/資深）；**衛教成效語**（「達最佳保養效果」）與**問句框架推薦**（「台南小兒科推薦：家長如何選擇？」＋客觀條件）合規；自稱式資格化禁。keywords／llms 可放 promotional。
2. 疫苗＝仿單＋醫師評估＋現貨來電確認＋停打但書；GLP-1 可見用泛稱＋off-label（既有 tirzepatide/Mounjaro/猛健樂衛教具名維持、不新增品牌標題）；首頁隱藏 GLP-1 品牌 5 條保留隱藏。
3. 禁自評 aggregateRating；評分數字不寫死。
4. 兒童肥胖：不節食不羞辱、健康體位三寶。
5. 不宣稱未具專科（耳鼻喉＝相關症狀＋轉介；家醫慢性病＝控制非治癒；明示非急診）。
6. 過敏檢測：「不單以數值忌口」但書。
7. 師承客觀陳述、勿暗示導師在本院執業或背書。
8. 修改後驗證：JSON-LD json.loads／標籤平衡／可見禁語掃描（移除隱藏層）／(視需要)Playwright。
9. **dateModified 完整 ISO 8601 含 +08:00；datePublished/lastReviewed 僅實際發布/審閱才動。**（本批⑤已將全站 dateModified/lastReviewed 純日期補足時區；datePublished/datePosted 純日期 30 處未動、待院長決定是否一併規範。）
10. 懶人包圖 alt/figcaption 中性衛教描述。
11. **（本批④新增作業原則）外部實體 sameAs 須逐一核對條目正確性**——誤連疾病實體比不連更糟；非標準診斷/症狀/非疾病實體不硬連。

## 八、文件維護協議（跨 session 接續機制；2026-07-06 制定）

> 目的：讓**任何**未來 session（不論模型強弱）都能安全地更新本套文件，不回退、不分岔、不越權。與 01 的「防呆」互為表裡：01 管做事，本節管改文件。

### 8-1 文件角色與權威順位
- **00（本檔）＝單一事實來源**。結構固定為：前八節＝**現況**（每批結束更新至最新）；文末「附：本批（日期）」＝**歷史**（append-only）。
- **衝突裁決**：①現況節與歷批附錄不一致 → 以**日期最新的批次附錄**為準，並回修現況節；②00 與 01 不一致 → 以 00 為準，回修 01（01 永遠是 00 的濃縮投影）；③00 與院長口頭指示不一致 → 院長指示為準，**但必須當批寫回 00**，否則下個 session 會照舊文件做錯。
- **AI 客服知識庫**：正本（155 題）由院長專案維護；Claude 不重產全文，只出「更新註記」檔，避免雙版本分岔。

### 8-2 誰可以改什麼
| 類別 | 未來 session 可自主 | 需院長核可 | 絕不可動 |
|---|---|---|---|
| 00 前八節現況更新（技術狀態、待辦勾銷附證據、錯字） | ✅ | | |
| 文末新增本批附錄、追加教訓 | ✅ | | |
| **第四節合規規則**任何增修 | | ✅（逐字） | |
| **待決事項**狀態變更（專業8處／5.0★／GLP-1隱藏5條／datePublished 30處） | | ✅ | |
| 任何標註「**裁示**」的內容 | | ✅ | |
| **歷批附錄既有文字** | | | ❌（只能在其後 append「勘誤（日期）：…」） |

### 8-3 更新格式（每批結束依序做）
1. 前八節現況同步（改到哪、寫到哪）。
2. 文末 append「附：本批（YYYY-MM-DD）」：改了什麼／為什麼／dateModified 政策（跳或不跳＋理由）／部署與驗證結果／新教訓。
3. 檔頭版本行更新日期＋一句摘要。
4. 01 同步（狀態段＋待辦段）。
5. **教訓寫法（固定三段式）**：現象 → 根因 → 一句可執行規則。
   - 正例：「visit-guide 急症框變灰（現象）→ `bg-red-50` 被 tailwind purge（根因）→ 新 class 頁面須納 content glob 或 append（規則）」。
   - 反例：「要小心 CSS 問題」——無現象無根因無動作，等於沒寫。

### 8-4 備份與長度管理
- **改 00／01 前必先** `cp` 為 `archive/檔名_backup_YYYYMMDD[abc].md`（接續包內留最近 2 份，更舊可刪）。
- 00 全檔超過約 **1,200 行**、或歷批附錄超過 **8 批**時：把最舊批次壓成 3–5 行精華（**只保留裁示、教訓、數字結論；刪執行細節**），並註明「完整版存於 lihsin接續包_該日期.zip」。壓縮前原文先進 archive/。

### 8-5 Session 交接模板（session 將盡或任務中斷時，貼進交付訊息並擇要寫回 00 第六節）
```
【交接 YYYY-MM-DD】
已完成：…（檔案路徑）
未完成：…（卡在哪一步、最後一次嘗試的錯誤原文）
```
